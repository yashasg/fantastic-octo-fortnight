### Key Findings

1. **Focus Mode Detection (INFocusStatusCenter)**
   - iOS 15+ API provides boolean `isFocused` state push-based observer
   - Cannot distinguish specific Focus profiles (Game vs Work vs Personal all collapse to `isFocused=true`)
   - Requires `NSFocusStatusUsageDescription` in Info.plist
   - Battery cost: negligible (push-based, zero polling)

2. **Critical App Detection (Maps/Games) — Hard Constraint**
   - No public API to enumerate foreground apps or detect another app's state
   - Private APIs (LSApplicationWorkspace, etc.) trigger App Store rejection
   - Solution: Use proxy signals (CarPlay, driving activity) for navigation context

3. **Proxy Signals**
   - CarPlay detection via `AVAudioSession.currentRoute` (no special entitlement)
   - Driving activity via `CMMotionActivityManager` (motion coprocessor, negligible battery)
   - Combination strategy: CarPlay OR driving = likely Maps/navigation session

4. **Architecture: PauseConditionManager**
   - Three protocol-backed detectors (FocusStatus, CarPlay, Driving)
   - Single aggregated boolean `isPaused` with observer callback
   - Testable via mock injection
   - Integrates with AppCoordinator snooze logic
